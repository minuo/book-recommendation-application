# 文档命名规则

## 1. 命名原则
- 清晰准确：文件名应清晰反映文档核心功能或主要内容
- 简洁明了：文件名不易过长，避免冗余词汇
- 一致性：所有文档应遵循统一的命名规范

## 2. 命名格式
- 使用小写字母
- 使用连字符（-）作为单词分隔符，不使用下划线或空格
- 文件名结构：`[描述性名称]-[文档类型]-[版本号（可选）].md`

## 3. 文档类型后缀
- `-documentation` 或 `-docs`：完整的API或系统文档
- `-design`：设计文档
- `-implementation`：实现方案文档
- `-guide`：指南或教程
- `-spec`：规范文档

## 4. 版本控制
- 当文档存在迭代更新时，在文件名末尾添加版本号
- 版本号格式：`-v[主版本号].[次版本号]`（如 `-v1.0`、`-v2.1`）

## 5. 示例
- `api-documentation.md`：API文档
- `api-design.md`：API设计文档
- `technical-implementation-plan.md`：技术实现方案
- `user-guide-v2.1.md`：用户指南（版本2.1）

## 6. 使用注意事项
- 避免使用缩写，除非是广泛认可的技术缩写（如API、URL）
- 保持文件名的稳定性，避免频繁更改
- 更新文档时，如内容有重大变更，应更新版本号
